## 社区生态概览

### 亚马逊云科技开发者成长旅程

作为一个活力满满的全球开发者社区，亚马逊云科技开发者社区中活跃着许多热情积极的社区领袖和技术专家。开发者们在社区中通过各种内容形式广泛分享他们的亚马逊云科技知识，包括博客文章、视频演讲、开源贡献、社区建设、开发者聚会等等。

面向活跃的开发者群体，亚马逊云科技为开发者提供完整的社区建设策略和技术支持。如果把开发者生态比作一个金字塔，我们对不同阶段的开发者都有不一样的策略和项目来提供支持：

- **亚马逊云科技 Hero** - 全球顶尖的亚马逊云科技技术专家
- **亚马逊云科技 Community Builders (CB)** - 亚马逊云科技技术爱好者，和热衷于知识分享并渴望与社区建立联系的开发者
- **亚马逊云科技 UserGroup (UG)** - 为开发者们提供学习和技术分享的社区组织
- **亚马逊云科技 Developers** - 使用亚马逊云科技技术和服务进行构建的开发者

## Hero项目

### 项目简介
亚马逊云科技 Hero 项目旨在发现和认可在亚马逊云科技开发者社区中熟悉亚马逊云科技的产品和服务、积极参与社区分享和贡献、最具影响力和号召力的技术专家，并和他们一起共同提高亚马逊云科技在社区和用户中的影响力。

### Hero 类别
基于亚马逊云科技的技术优势和开发者的兴趣方向，Hero 分为以下几个类别：

1. **Community Heroes** - 社区建设专家
2. **Container Heroes** - 容器技术专家
3. **Machine Learning Heroes** - 机器学习专家
4. **Serverless Heroes** - 无服务器计算专家
5. **Data Heroes** - 数据技术专家
6. **Security Heroes** - 安全技术专家
7. **DevTools Heroes** - 开发工具专家

### 全球分布
- 全球共有238位亚马逊云科技 Hero 分布于55个国家和地区
- 大中华地区共有16位 Heroes，其中9位来自中国大陆地区

## User Group项目

### 3.1 User Group项目简介

亚马逊云科技 UG 是一个以社区为导向，不断发展的组织，将对亚马逊云科技的云计算服务感兴趣的个人和专业人士聚集在一起，是亚马逊云科技相关知识的分享和学习平台，定期组织聚会、研讨会、或者网络研讨会等。

#### 核心特点
- **独立运作** - 不归属于亚马逊云科技，是独立运作的用户和开发者社区
- **社区驱动** - 活动组织由社区驱动，根据地区和当地组织者的不同而有所变化
- **技术聚焦** - 围绕特定的亚马逊云科技服务、行业技术应用，或一般亚马逊云科技的技术讨论
- **全球网络** - 全球有470+ UG，覆盖了100+国家，有720+ UG负责人和50万以上的积极会员

#### 中国UG发展历程
- **2021年** - 重新规划，从一个UG开始
- **2024年** - 发展成为全国12个UG的规模（10个正式UG，2个试运营UG），有28个UG负责人

### 3.2 UG 2024全新目标

2024年亚马逊云科技UG将迎来新的改版，将会成立第一届UG组委会，之后UG相关的运营决策将由组委会在符合UG规章制度条件下，进行决策和执行。

#### 核心重点
1. **独立运营** - 由亚马逊云科技UG组委会全权组织，协调，和执行
2. **专业化发展** - 增加会员中亚马逊云科技专业用户开发者数量，目标从10%提升到80%，2024年新用户中30%为专业开发者
3. **深度内容** - 提供深度内容，支持用户的学习和成长，通过维护专业用户讲师群来提升内容深度
4. **品牌建设** - 宣传亚马逊云科技UG的品牌，增加UG在用户/合作伙伴中的曝光
5. **规模增长** - 实现用户组、用户人数，及参与活动人数（线上和线下）的增长

### 3.4 负责人 (UGL)

#### UGL职责
- 担任UserGroup成员的主要联系人
- 每年至少举办四次活动（每季度一次），可以是线上活动，也可以是线下活动
- 使用亚马逊云科技官方活动平台开展所有活动，并通过UserGroup自己的社交媒体渠道进行宣传
- 为活动确定主题、招募发言人并寻找场地
- 与亚马逊云科技UserGroup对接人保持联系，参加月度电话会议，定期分享会员反馈

#### UGL类型

**1. 城市组织者**
- 要求对亚马逊云科技某一行业、或某一领域技术有热情、有理解
- 愿意为UG/亚马逊云科技社区做出贡献
- 积极与本地社区进行深入合作，组织线下/线上活动

**2. 技术出品人 (Track Owner)**
- 可以是亚马逊云科技员工、Hero、Builder或来自社区的资深技术专家
- 主要责任是定义技术活动方向，主题，和内容框架
- 技术出品人小组，希望能为每个技术专题确定4位左右的专家

**3. 技术专家演讲者 (Speaker)**
- 可以是亚马逊云科技员工、Hero、Community Builder或来自客户/合作伙伴/社区的资深技术专家
- 每位专家会标识有擅长的话题方向
- 要求对具体演讲主题有足够了解，可以进行面向开发者的代码开发演讲

#### UGL福利
- 发展您的专业品牌、技能组合、人际网络和领导技能
- 在亚马逊云科技用户中获得认可和声誉
- 在亚马逊云科技开发者社区中添加特别表彰和数字徽章
- 提供免费的亚马逊云科技认证考试，扩展您的专业能力（每年活跃的UserGroup社区组织者提供认证考试一次）
- 与全球其他的UserGroup负责人进行沟通
- 参加专为负责人举办的大会和其他活动
- 获得组长专属品牌礼品和活动机会
- 颁发年度卓越UserGroup社区组织者的认证
- 最活跃的UserGroup社区的组织者将有机会获得美国re:Invent大会免费门票

#### 注意事项
1. **赞助管理** - UG可以对接赞助，UGL可以自行联系，但最后要由组委会统一进行评估和协商
2. **费用管理** - 赞助费用由组委会统一管理，赞助商可以指定参与某场具体的活动
3. **数据保护** - UGL不允许将活动报名PII数据交与赞助商，我们将提供二维码扫码器，帮助赞助商收集参会者信息

### 技术主题方向

#### 1. GenAI（生成式AI技术）
生成式AI技术作为人工智能技术的重要分支，正在逐步得到广泛的应用和推广。亚马逊云科技在生成式人工智能领域展现出卓越的技术优势：

**核心技术：**
- **Amazon Inferentia** - 自研的机器学习推理芯片，专为处理大规模生成式AI模型任务而设计
- **Amazon Trainium** - 训练芯片，通过高度优化的硬件架构和先进的深度学习功能，使得GenAI开发者能够更加高效地进行模型训练
- **Amazon Titan** - 通过集成先进的深度学习技术，为用户提供了强大而灵活的生成式AI基础模型
- **Amazon BedRock** - 支持多种生成式AI的基础模型，使得开发者能够更加灵活地选择和部署适合其需求的基础模型

**安全与合规：**
- 提供多种支持安全和隐私要求的服务（HIPAA资格和符合GDPR合规）
- Amazon BedRock拥有全生命周期数据治理方案
- 数据在传输和静态时始终处于加密状态
- 帮助用户构建负责任的AI

#### 2. 大数据
大数据和人工智能技术高速发展，成为新一轮科技革命和产业变革的核心驱动力。亚马逊云科技提供了一系列强大的云上大数据技术：

**服务组合：**
- **Amazon S3** - 数据存储
- **Amazon EMR** - 大数据处理
- **Amazon Redshift** - 数据仓库
- **Amazon SageMaker** - 机器学习平台

**技术优势：**
- 广泛的服务组合，涵盖数据存储、处理、分析和机器学习等方面
- 高度的可扩展性，可以根据业务需求弹性调整计算和存储资源
- 安全和合规性，提供高水平的数据安全性，满足各种合规性要求
- 智能化服务，引入了众多人工智能和机器学习服务

#### 3. 云原生
亚马逊云科技为云原生开发提供了全面的解决方案，支持开发者构建高效、可靠、安全的云原生应用：

**核心优势：**
- **全球基础设施** - 拥有全球广泛的基础设施，数据中心网络覆盖多个大陆
- **容器服务** - 提供强大的容器服务，如Amazon ECS和EKS，支持使用Docker容器进行应用程序的打包和部署
- **无服务器计算** - Amazon Lambda服务支持无服务器计算，使开发者能够以事件触发的方式执行代码
- **监控和安全** - 全面的监控、日志和安全服务，通过CloudWatch、CloudTrail等服务确保应用的稳定性和安全性

## 活动形式规划

### 线下活动

#### Community Day
- **规模**：300人左右
- **形式**：全天线下活动
- **受众**：技术爱好者，专业开发者，TDM
- **频次**：2024年预计举办1场活动
- **特色**：以技术分享、讨论和动手实验室为特色，由来自世界各地的技术专家，亚马逊云科技Hero，UserGroup负责人或行业领袖来分享

#### Meetup
- **规模**：50~100人
- **形式**：半天的线下活动
- **频次**：2024年预计举办28场活动
- **类型**：
  - **Workshop** - 半天或大半天时间，由4-5个主题分享的活动类型
  - **Hands-On** - 半天或者大半天时间，由开发者自带电脑，在老师指导下完成动手项目的活动形式
  - **Panel Discussion** - 半天或者大半天研讨会类型，活动嘉宾要求为大咖，自带流量

### 线上活动

#### NightTalk
- **时间**：每周四晚8:00~9:00（非假期）
- **形式**：为社区开发者设计的轻量/高频技术分享系列线上活动
- **内容**：每周一个全新的系列主题，由开发者对自己感兴趣的技术话题进行投票
- **受众**：技术爱好者和TDM
- **频次**：2024年预计举办15场活动
- **特点**：4-8场分享为一期，由具体技术出品人负责一期的内容制定和讲师邀请，只针对会员，不做对外宣传

#### Tech Talk
- **主导**：亚马逊云科技主导的系列线上内容分享
- **受众**：面向对云计算、AI、IoT等技术感兴趣的开发者
- **内容**：涵盖不同技术水平的内容，由亚马逊云科技、合作伙伴或客户的技术专家进行分享
- **特色**：包含现场演示与互动问答环节
- **频次**：2024年预计完成15场活动

### 特别活动主题

#### IWD (International Women's Day)
- **时间**：每年3月-4月
- **目标**：特别面向女性开发者的社区分享活动
- **宗旨**：打造一个没有偏见、定型观念和歧视的世界，拥抱公平
- **内容**：邀请精彩的女性开发者和职场人分享她们的故事，了解她们的职业道路、成功经验和面对挑战的勇气

#### re:Invent Recap
- **背景**：亚马逊云科技re:Invent全球大会是全球云计算引领者举办的年度盛会
- **时间**：每年12月至来年1月
- **内容**：各个地区会举办面向用户和开发者的系列活动分享并解读re:Invent上的精彩发布
- **形式**：UserGroup社区技术专家团队整理相关re:Invent大会技术内容，由UserGroup负责人组织相关线下沙龙活动

## 申请流程详解

### Hero申请流程
1. **内部推荐** - 需要现有Hero或亚马逊云科技员工推荐
2. **在线申请** - 访问 https://dev.amazoncloud.cn/ug/form/hero
3. **材料审核** - 提交个人技术背景和社区贡献材料
4. **面试评估** - 与Hero项目团队进行面试
5. **结果通知** - 通过评估后正式成为Hero

### Community Builder申请流程
1. **申请时间** - 每年一月前后开放申请
2. **在线申请** - 访问 https://aws.amazon.com/cn/developer/community/community-builders/
3. **材料提交** - 提交个人技术背景和社区参与经历
4. **审核评估** - CB项目团队进行审核
5. **结果通知** - 通过后加入CB社区

### UGL申请流程

#### 1. 在线申请
登录到 https://dev.amazoncloud.cn/community，申请成为领导者

#### 2. 材料准备
在申请表格中提供：
- 个人信息（姓名、联系方式、职业）
- 相关技术背景和社区活动经验
- 申请动机和未来计划

#### 3. 面试评估
组委会安排面试，评估申请人的能力和适合度

**面试问题示例：**

**个人背景类：**
- 请简述一下你自己，包括你现在所在的公司，title，技能、经验和背景
- 在你的职业生涯中，有哪些成就是你最为骄傲的？
- 你如何处理工作压力？你有哪些应对策略？

**领导力类：**
- 作为一名用户组领导，你认为最重要的职责是什么？
- 如果你所在社区进行一次重要的活动，你会如何策划和准备？
- 你认为在团队中培养领导力最重要的因素是什么？

**技术和社区类：**
- 谈谈你对亚马逊云科技有什么样的了解？谈谈你最熟悉的亚马逊云科技服务
- 你如何吸引到更多有亚马逊云科技背景的成员参与进UG？

#### 4. 结果通知
通过面试和评估后，正式成为UGL

### 志愿者申请流程

#### 方式一：在线注册
1. 点击 https://dev.amazoncloud.cn/ug/form/lecturerRegister 登录界面
2. 点击跳转登录页面
3. 登录审核通过的账号
4. 完成志愿者注册

#### 方式二：活动报名
1. 根据活动形式找到开展志愿者注册工作的团组织负责人
2. 直接报名志愿者
3. 由当地活动负责人对申请人进行审核
4. 审核合格并参与活动结束后，向申请人颁发注册志愿者证章

**志愿者报名方式：**
1. **与UGL联系** - 了解到UGL正在为某项活动招募志愿者后，直接与他们取得联系
2. **提交申请** - 根据UGL的要求，填写志愿者申请表
3. **参加面试或培训** - 提交申请后，UGL可能会安排面试或培训
4. **获得志愿者证书** - 完成志愿服务后，UGL会提供认证可查询的志愿者证书
